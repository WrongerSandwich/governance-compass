# Map Spec — Synthetic Study

Implementation spec for the world map element used on the Synthetic Study section of governance-compass.org. Covers the interactive choropleth on the Personas page and the static choropleths on the Patterns page.

## Scope

Two variants of the same base map:

- **Interactive** — Personas page. Click a region to filter the persona browser below.
- **Static** — Patterns page. Display-only, three distinct analytical views.

Both variants share base geometry, region taxonomy, color treatment, and the transnational / thinness handling described below.

## Region taxonomy

The synthetic study's 1,002 personas were generated by Gemini with a custom 10-region scheme baked into each persona's `region` field. Use this scheme as-is — do not attempt to retrofit UN M49 or World Bank regions.

| Region key | Persona count | Geographic? |
|---|---|---|
| `western_europe` | 134 | Yes |
| `south_southeast_asia` | 133 | Yes |
| `east_asia` | 112 | Yes |
| `latin_america` | 112 | Yes |
| `eastern_europe_central_asia` | 111 | Yes |
| `middle_east_north_africa` | 111 | Yes |
| `sub_saharan_africa` | 111 | Yes |
| `north_america` | 89 | Yes |
| `oceania_small_states` | 33 | Yes |
| `diaspora_transnational` | 56 | **No** |

### Transnational handling

The `diaspora_transnational` region has no geographic referent — its personas live scattered across many host countries but are tagged as transnational because that is their identity, not their location. Do not place them on the map geography.

Render a small legend tile adjacent to the map, labeled **"Transnational (56)"**, visually echoing the region treatment (same shape, same palette). The tile is clickable on the interactive map and functions identically to a region click: it selects the 56 transnational personas as a filter. On static maps the tile is present but non-interactive, with the same density shading used for the geographic regions.

## Library and base geometry

**Library:** `react-simple-maps@3`. Thin `d3-geo` wrapper, ~30kb gzipped, plays cleanly with Next.js App Router, SVG output that respects CSS variables. No tile server dependency.

**Base geometry:** Natural Earth 110m, served as TopoJSON. Derive a region-level TopoJSON by aggregating country polygons into the 10-region scheme; store both the country-level and region-level files in `/public/geo/`. Country polygons are needed only for the optional country-level shading described below.

Country-to-region mapping table should be checked into the repo so the aggregation is reproducible and auditable.

### Country name normalization

The raw persona data contains inconsistent country labels (`USA` vs `United States`, `Cote d'Ivoire` vs `Côte d'Ivoire` vs `Ivory Coast`, `DR Congo`, malformed entries like `"Italy (rural village)"`, `"Syria)"`). Before any country-level work:

1. Add a normalized `country_iso` field (ISO 3166 alpha-3) to the persona data as a preprocessing step.
2. Handle the malformed entries explicitly — either repair via inspection or drop the parenthetical/trailing-paren content.
3. The raw `location` field stays as-is for display purposes; `country_iso` is for mapping only.

## Color treatment

All fills use CSS variables derived from the warm stone palette. No direct hex values in components.

### Density shading (Persona density map)

Five-step scale from lightest to darkest:

- `--map-density-0`: Stone 100 or similar near-white (very faint)
- `--map-density-1`: Stone 200
- `--map-density-2`: Stone 400
- `--map-density-3`: Stone 600 (the primary brand tone, `#85735e`)
- `--map-density-4`: Stone 800

Map each region's persona count to a bin via quantile or fixed-breakpoint scale. With 10 regions and a spread from 33 to 134, fixed breakpoints work fine: `[<50, 50–89, 90–112, 113–133, 134+]` maps to density 0–4. The thinnest region (Oceania, 33) sits at density 0; the densest (Western Europe, 134) sits at density 4.

### Borders and states

- Region borders: `--map-border` (Stone 400, 0.5px)
- Hover fill: `--map-hover` (Stone 800) with 1px outline in `--map-accent` (warm gold `#8B6914`, the glossary-highlight color — this creates an echo across site elements)
- Selected fill: holds hover fill plus a 1.5px gold outline
- Unselected (when another region is selected): 40% opacity of original fill

### Country-level shading (optional sub-layer, Persona density map only)

Countries with **n ≥ 10 personas** render a faint country-level overlay on top of the region fill, using the same five-step density scale at 40% opacity. Countries below the threshold do not render any country-level overlay — the region fill shows through unchanged. This gives the map a "zoom in with your eye" quality without promising granularity the data can't support.

Based on the current dataset, this surfaces ~28 countries: Mexico (35), Canada (32), China (29), USA (24), Japan (21), France (19), South Korea (19), India (19), Germany (18), UK (16), Taiwan (16), Indonesia (14), Thailand (14), Philippines (13), Vietnam (13), Bangladesh (12), Sri Lanka (12), Egypt (12), Italy (11), Mongolia (11), Pakistan (11), South Africa (11), Sweden (10), Lebanon (10), Kenya (10), Ghana (10), Ethiopia (10).

## Thinness treatment

Any region with zero personas in a given view (e.g., Oceania in a cluster-specific filter) renders as `--map-density-0` (the near-white), not as "no fill" or a default gray. Hover tooltip includes the count explicitly, e.g. `"n=0 in this view"`.

Same for country-level shading: countries below n=10 render nothing at country level; the region fill is what the user sees.

## Interactive map — Personas page

### Layout and placement

- Top of the Personas page, above the facet filters and persona grid.
- Width: full content width (match the facet filter block below).
- Aspect ratio: ~2:1 for desktop (wide enough for a world projection), ~16:10 for mobile.
- Transnational legend tile: positioned adjacent to the map on desktop (right side), below the map on mobile.

### Projection

`geoNaturalEarth1` from `d3-geo-projection`. Readable world view, minimal polar distortion, no zoom/pan.

### States

**Default.** Full world, regions shaded by persona density. Transnational tile shaded by its own density. No selection.

**Hover.** Region fills to `--map-hover`, outline in `--map-accent`. Tooltip appears near cursor:
```
Western Europe
134 personas
Top 3 archetypes: [archetype], [archetype], [archetype]
```
Countries with n ≥ 10 get their own hover tooltip on country hover:
```
France
19 personas
```
Country hover does not filter — only regions and the transnational tile are clickable.

**Selected.** Clicked region holds hover fill + gold outline. All other regions (including the transnational tile, if unselected) dim to 40% opacity. Facet filter state updates to filter by that region. URL updates with `?region=<region_key>`. A "Clear region filter" affordance appears above the facet filters.

**Clear.** Clicking the selected region again, clicking the clear affordance, or removing `region` from the URL reverts to default state.

### Interaction constraints

- No zoom, no pan.
- Clicking a region is a discrete filter action, not spatial navigation.
- Only one region can be selected at a time (click a different region = switch selection, not multi-select).
- The transnational tile behaves identically to a region click.

### URL state

`?region=<region_key>` where `region_key` is one of the 10 region keys above. URL is the source of truth for filter state. Bookmarking or sharing a URL preserves the region filter. Cross-page navigation within Synthetic Study preserves the filter (per IA decision); leaving Synthetic Study clears it.

### Mobile

- Full-width, ~16:10 aspect.
- Scroll-lock behavior: map does not capture vertical scroll; users scrolling past the map should not get stuck on it.
- Tooltips on tap: tapping a region shows the tooltip content in a persistent element below the map (not a floating tooltip) AND selects the region in the same action. No two-step peek-then-commit.
- Transnational tile: below the map, full-width or half-width, tap to select.
- Countries: no hover layer on mobile. Country-level shading still renders but is display-only.

### Accessibility

- Each region has `aria-label="<Region name>, <count> personas, <dominant archetype>"`.
- Tab order cycles through regions in the order they appear in the source (roughly alphabetical by key, or explicit ordering).
- Enter/space on a focused region selects it; Escape clears selection.
- Screen-reader summary `aria-description` on the map element: total persona count, densest region, presence of the transnational tile.
- Color is never the only signal — selected state has both a color shift and a gold outline; the density scale is paired with counts in all tooltips and labels.
- Density-0 (zero or near-zero) is distinguishable from density-1 via a subtle border weight difference as well as color, for colorblind safety.

## Static maps — Patterns page

Same base geometry, same palette family, same transnational tile. No click handlers. Hover tooltips only (on desktop; tap-to-peek on mobile, no selection).

Each static map has a prose paragraph adjacent (not below as a caption — sitting alongside) pointing at what's visible. Prose does not restate what's on the map; it points.

### Map 1 — Persona density by region

Visually identical to the Personas-page default state. Deliberate echo: the reader recognizes "this is the same map, used differently here." Serves as the anchor the other two reference.

Position: first map in the regional aggregates section.

### Map 2 — Dominant cluster by region

Six clusters, six fills. Each region fills to the color assigned to its dominant cluster (most common cluster among that region's personas).

Dominant cluster by region (from the scored dataset, 1,002 personas assigned via nearest centroid on averaged-across-models profiles):

| Region | Dominant cluster | Share |
|---|---|---|
| Western Europe | C2 | 38% |
| Eastern Europe / Central Asia | C2 | 37% |
| MENA | C5 | 36% |
| North America | C4 | 36% |
| Oceania small states | C3 | 33% |
| sub-Saharan Africa | C1 | 32% |
| East Asia | C5 | 31% |
| Diaspora transnational | C0 | 30% (near-tie with C4 at 30%) |
| Latin America | C1 | 29% |
| South/Southeast Asia | C1 | 21% |

All six clusters appear as dominant somewhere — the map will be visually varied, not monochromatic.

**Dominance strength varies.** Western Europe at 38% is meaningful dominance; S/SE Asia at 21% is barely a plurality. Render dominance strength as a secondary visual signal — a subtle opacity attenuation or hatching for regions below ~25% dominance. Specifically:

- Dominance ≥ 30%: full saturation
- Dominance 25–29%: 80% opacity of the cluster fill
- Dominance < 25%: 60% opacity + a fine diagonal hatching pattern

Prose alongside the map should name this explicitly — "where regions shade lightly or show hatching, no single cluster has a clear majority."

Diaspora tile: render at 60% opacity with hatching (C0 vs C4 essentially tied).

### Cluster color assignments

Map cluster thematic centers to the existing four domain colors, plus two derived variants for the remaining clusters:

| Cluster | Character | Color |
|---|---|---|
| C2 | Collective provision + pluralism + progressive (dominant in Europe) | **Stone** |
| C5 | Growth + centralized + cohesion (dominant in East Asia, MENA) | **Slate** |
| C3 | Distributed governance + communitarian (dominant in Oceania) | **Sage** |
| C1 | Collective provision + non-interventionist (dominant in Latin America / SSA / SSEA) | **Clay** |
| C0 | Pragmatic centrist (most muted) | **Stone 300** (desaturated variant of Stone — signals "central / balanced") |
| C4 | Radical-egalitarian strong form (dominant in North America) | **Clay-rust** (warmer / more saturated variant of Clay — C4 is thematically close to C1, so adjacency in the palette maps to adjacency in the data) |

The visual distance between C1 and C4 colors should be proportional to their thematic distance (adjacent in the radical-egalitarian family). Variants-of-the-same-hue, not two unrelated colors.

Legend below the map pairs each cluster color with its ID and a short descriptor (e.g., "C2 — Progressive pluralist"). Full cluster descriptions live in the cluster characterization section above the map, not in the legend.

### Map 3 — Axis 8 gradient (Cultural Diversity)

Single axis gradient. Axis 8 chosen because it shows the strongest and cleanest regional variation (range 0.80 across regions, clean positive-to-negative spread with a coherent narrative).

Regional mean scores:

| Region | Mean axis 8 |
|---|---|
| Eastern Europe / Central Asia | +0.43 |
| East Asia | +0.27 |
| Western Europe | +0.23 |
| MENA | +0.19 |
| sub-Saharan Africa | −0.05 |
| S/SE Asia | −0.02 |
| Oceania small states | −0.10 |
| North America | −0.15 |
| Latin America | −0.23 |
| Diaspora transnational | −0.37 |

**Palette:** diverging, centered on 0. Warm tone (muted terracotta / clay) for positive (Cohesion), cool tone (muted slate) for negative (Pluralism), with a neutral pale stone at the midpoint. Five or seven bins.

The two endpoint labels on the legend: **Pluralism** (negative) and **Cohesion** (positive), matching the axes page terminology exactly.

Prose alongside names what's visible (Eastern Europe/Central Asia and East Asia toward cohesion; Diaspora and the Americas toward pluralism) without overclaiming causation.

### What's explicitly not included

- **Dominant-archetype-by-region map.** 12 archetypes is too many colors for a legible choropleth, and 6 archetypes are empirical-zero so the map would really only use 6 colors anyway — duplicates Map 2. Archetype distribution lives in its own Patterns subsection, not as a map.
- **Gradient maps for other axes.** One gradient earns its real estate; two starts to feel like a dashboard. If readers want per-axis breakdowns, the download is for them.
- **Axis 11 (Military Policy) as a gradient** despite its 0.74 range — every region is negative, so the pattern is "everyone's non-interventionist, with degrees," which is prose material, not map material.

## Mobile treatment (static maps)

- Stack vertically, one per section.
- Same tap-to-peek behavior as interactive map, minus the selection side effect. Tapping reveals the tooltip in a persistent element below the map; tapping elsewhere or the map background dismisses.
- Prose appears below the map on mobile rather than alongside.
- Legends are horizontal-scroll on mobile if the color swatches would otherwise wrap awkwardly.

## Performance notes

- TopoJSON files should be served as static assets, gzip-compressed, with long cache headers.
- Map renders on the client after hydration; initial SSR can ship a static SVG snapshot for the default state (no selection) to avoid layout shift.
- Tooltip rendering should be debounced on mouse-move to avoid thrashing on rapid cursor movement across regions.

## Open items (non-blocking)

- **Country-to-region mapping table** needs to be authored before country-level shading can render. Every country that appears in the persona data needs exactly one region assignment. Edge cases (e.g., does Turkey belong to MENA, Eastern Europe/Central Asia, or neither in this scheme?) should be resolved by checking how Gemini actually tagged personas from that country during generation — the `region` field on each persona is authoritative.
- **Final CSS variable names** can follow existing site conventions. The names above are illustrative.
- **Cluster color tokens** (`--cluster-0` through `--cluster-5`) should be defined at the same level as the domain color tokens so they're usable in other Patterns components (cluster radar thumbnails, cluster characterization cards) without redefinition.
