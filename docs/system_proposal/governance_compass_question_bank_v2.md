# The Governance Compass — Question Bank v2

## Revision Notes

This is a full revision of the question bank adding two layers of progressive disclosure:

**Forced-choice items** now have a **headline** (6–12 words, displayed at 15px weight 500) above the **body text** (displayed at 13px, text-secondary color). The headline carries the core claim; the body provides nuance. Users can decide based on headlines alone or read the body for precision.

**Scaled items** now have **short labels** (2–5 words per option, displayed in the segmented bar) with **detail text** (shown below the bar when an option is selected). The short labels make the gradient instantly scannable; the detail text confirms what each position means.

### Structure Per Item

**Forced-Choice:**
```
[Headline A]                    [Headline B]
[Body text A in lighter style]  [Body text B in lighter style]
```

**Scaled:**
```
[Question stem]
[ Label 1 | Label 2 | Label 3 | Label 4 | Label 5 ]
  Pole A ←                                    → Pole B
[Detail text for selected option appears here]
```

### Scoring

Unchanged from v1. Headlines and short labels are display-only — they don't affect scoring. The underlying item logic is identical.

## Implementation Notes Addendum

### Rendering the Two-Layer Text

**Forced-choice cards:**
```
┌──────────────────────────────┐
│                              │
│  [Headline]                  │  ← 15px, weight 500, text-primary
│  [Body text]                 │  ← 13px, weight 400, text-tertiary
│                              │
└──────────────────────────────┘
```

The body text should be visually subordinate — a reader who only scans the headlines should still get the gist of each position. The body text is available for anyone who wants to confirm the nuance matches their interpretation of the headline.

**Scaled segmented bar:**
```
[ Label 1 | Label 2 | Label 3 | Label 4 | Label 5 ]
  Pole A ←                                    → Pole B
┌─────────────────────────────────────────────────────┐
│  [Detail text for the selected option]              │
└─────────────────────────────────────────────────────┘
```

The detail text block appears only when an option is selected. If no option is selected, no detail text is shown. If the user changes their selection, the detail text updates to match the newly selected option. The detail block uses a subtle background (Surface 2) to visually distinguish it from the bar itself.

### Headline Writing Guidelines (For Future Item Revisions)

- 6–12 words. Under 6 feels like a bumper sticker; over 12 starts requiring parsing.
- Must be a complete claim, not a fragment. "Protect individual privacy" is a fragment. "Better that some threats go undetected" is a claim with a position built in.
- The two headlines in a forced-choice pair should be syntactically parallel where possible. If one starts with "The state should...", the other should too. Parallel structure makes comparison faster.
- Avoid negations in headlines. "The state should not restrict technology" is harder to parse than "Technology should be free to develop." The brain processes affirmative claims faster.
- Headlines should make the core tension visible without reading the body. If you cover the body text and can't tell what the dilemma is from the headlines alone, rewrite them.

### Short Label Writing Guidelines (For Future Item Revisions)

- 2–5 words. These sit inside segmented bar segments that may be as narrow as 100px on mobile.
- Must form a visible gradient when read left to right. The reader should be able to see the spectrum without reading detail text.
- Symmetric phrasing where possible. "Strictly limited / Somewhat limited / Balanced / Broad / Expansive" reads as a clear gradient. "Very restricted / Kind of open / It depends / Open / Totally free" does not.
- The middle option should feel like a genuine midpoint, not a cop-out. "Balanced," "Moderate," "Case by case," "Both equally" are all acceptable midpoint labels.
- Avoid the word "neutral" — it implies disengagement rather than a considered middle position.
